Data Structures

Stack ??LIFO Data Structure

A stack follows the Last In, First Out (LIFO) principle. It is used internally in DFS implementations.

Queue ??FIFO Data Structure

A queue follows the First In, First Out (FIFO) principle. Essential for BFS traversal.

Array

Elements stored in contiguous memory. The foundational structure for Binary Search and Two Pointer algorithms.

Linked List

Composed of nodes and pointers. Faster insertion/deletion compared to Arrays.

Hash Table

Key-value pair storage with O(1) average lookup time.

Heap / Priority Queue

Complete binary tree structure. Core data structure for Dijkstra's algorithm.

Binary Search Tree (BST)

Left < Parent < Right property. Traversable via BFS and DFS.

Algorithms

Bubble Sort

O(n²) comparison sort. Slower than Quick Sort.

Merge Sort

O(n log n) stable divide-and-conquer sort.

Quick Sort

Pivot-based O(n log n) sort. More cache-efficient than Merge Sort.

Binary Search

O(log n) search in a sorted Array.

BFS (Breadth-First Search)

Queue-based level-by-level traversal. Guarantees shortest path in unweighted graphs.

DFS (Depth-First Search)

Stack/recursion-based. Explores depth-first, unlike BFS.

Dijkstra's Algorithm

Heap-based shortest path in weighted graphs.

Greedy Algorithm

Makes locally optimal choices. Often compared with DP.

Dynamic Programming (DP)

Memoization-based optimization. Most frequently tested in coding interviews.

Two Pointer

O(n) search on sorted Arrays. Great combo with Binary Search.
